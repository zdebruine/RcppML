## Backwards Compatibility — How It Works

The `FileHeader_v2` has `uint8_t reserved[32]` at bytes 96–127 (zero-filled on
old files). The new design repurposes these bytes as typed fields **read by
accessor methods only** — the raw struct layout is unchanged:

```
reserved[0:7]   → obs_table_offset  (uint64_t, 0 = no obs table)
reserved[8:15]  → var_table_offset  (uint64_t, 0 = no var table)
reserved[16:19] → transp_chunk_cols (uint32_t, 0 = same as chunk_cols)
reserved[20]    → flags             (uint8_t,  bit 0 = has_dimnames)
reserved[21:31] → zero (future)
```

Because old files have `reserved` = all-zeros:
- `obs_table_offset == 0` → no obs table → `st_read_obs()` returns empty data.frame
- `var_table_offset == 0` → no var table → `st_read_var()` returns empty data.frame
- `transp_chunk_cols == 0` → use `chunk_cols` for transpose too
- `transpose_offset == 0` → no transpose stored → `st_slice_rows()` returns error
  with message: "File has no transpose section; re-write with include_transpose=TRUE"

**GEO reprocessing jobs currently running on the cluster generate v2 files with
`include_transpose=FALSE` (the old write default). These files will remain fully
readable for `st_read()` and `st_slice_cols()`. New writes default to
`include_transpose=TRUE`. Do not remove the `include_transpose` parameter.**

Add these accessor methods to `FileHeader_v2` in `header_v2.hpp` (after the
existing `valid()` method):

```cpp
// Accessors for reserved-byte extension fields (backwards-compatible read)
uint64_t obs_table_offset() const {
    uint64_t v = 0; std::memcpy(&v, reserved,     8); return v;
}
uint64_t var_table_offset() const {
    uint64_t v = 0; std::memcpy(&v, reserved + 8, 8); return v;
}
uint32_t transp_chunk_cols() const {
    // 0 means "same as chunk_cols"
    uint32_t v = 0; std::memcpy(&v, reserved + 16, 4); return v;
}
uint8_t flags() const { return reserved[20]; }
bool has_dimnames() const { return (flags() & 0x01) != 0; }

void set_obs_table_offset(uint64_t off) { std::memcpy(reserved,      &off, 8); }
void set_var_table_offset(uint64_t off) { std::memcpy(reserved + 8,  &off, 8); }
void set_transp_chunk_cols(uint32_t cc) { std::memcpy(reserved + 16, &cc,  4); }
void set_has_dimnames(bool v) {
    reserved[20] = v ? (reserved[20] | 0x01) : (reserved[20] & ~0x01);
}
```

**These are the ONLY changes to `header_v2.hpp`.** Do not modify any struct
sizes, field names, or existing methods.

## Phase 1 — Obs/Var Binary Column-Store

### 1.1 New header: `inst/include/streampress/format/obs_var_table.hpp`

Define the binary column-store format for embedded obs and var tables.

```cpp
// Sentinel values for missing data
static constexpr int32_t  NA_INT32   = INT32_MIN;
static constexpr uint32_t NA_UINT32  = UINT32_MAX;
static constexpr float    NA_FLOAT32 = std::numeric_limits<float>::quiet_NaN();
static constexpr double   NA_FLOAT64 = std::numeric_limits<double>::quiet_NaN();

enum class ColType : uint8_t {
    INT32       = 0,   // raw int32_t array; NA = INT32_MIN
    FLOAT32     = 1,   // raw float array;   NA = NaN
    FLOAT64     = 2,   // raw double array;  NA = NaN
    BOOL        = 3,   // raw uint8_t array (0/1); NA = 255
    UINT32      = 4,   // raw uint32_t array; NA = UINT32_MAX
    STRING_DICT = 5,   // uint32_t code array (NA = UINT32_MAX) + null-delimited dictionary
};

// Fixed-size column descriptor: 112 bytes
struct ColDescriptor {
    char     name[64];         // null-terminated column name; 63 chars max
    uint8_t  col_type;         // ColType enum
    uint8_t  nullable;         // 1 if column contains NA sentinels
    uint8_t  _pad[2];          // alignment
    uint32_t dict_bytes;       // byte count of null-delimited dict (STRING_DICT only; 0 otherwise)
    uint64_t data_offset;      // byte offset from start of table blob to raw data array
    uint64_t dict_offset;      // byte offset from start of table blob to dictionary blob
    uint8_t  _reserved[20];    // zero, future use
    // Total: 64+1+1+2+4+8+8+20 = 108 → pad to 112
    uint8_t  _align[4];
};
// static_assert(sizeof(ColDescriptor) == 112, "ColDescriptor must be 112 bytes");

// Table header: variable length
// Layout in file: [ObsVarTableHeader][ColDescriptor × n_cols][raw data blobs]
struct ObsVarTableHeader {
    uint8_t  magic[4];     // "OVTB"
    uint32_t n_rows;       // number of rows (cells or genes)
    uint32_t n_cols;       // number of data columns
    uint32_t header_bytes; // byte size of this header + all descriptors
    // ColDescriptor[n_cols] follows immediately
};
// static_assert(sizeof(ObsVarTableHeader) == 16, "ObsVarTableHeader must be 16 bytes");
```

Provide these C++ functions in the same header or a companion implementation file
(`obs_var_table_impl.hpp`):

```cpp
// Write an obs/var table to a byte buffer (returned as std::vector<uint8_t>)
// col_names: name of each column
// col_types: ColType for each column
// int_cols:  map of column index → std::vector<int32_t>
// flt_cols:  map of column index → std::vector<float>
// dbl_cols:  map of column index → std::vector<double>
// bool_cols: map of column index → std::vector<uint8_t>
// dict_cols: map of column index → pair<std::vector<uint32_t>,string_dict>
std::vector<uint8_t> obs_var_table_serialize(
    const std::vector<std::string>& col_names,
    const std::vector<ColType>&     col_types,
    ...);

// Read back to per-column vectors
void obs_var_table_deserialize(
    const uint8_t* buf,
    size_t buf_bytes,
    std::vector<std::string>& col_names_out,
    std::vector<ColType>&     col_types_out,
    ...);
```

The serializer and deserializer must be fully correct and round-trip lossless. Use
`pread`/`ReadFile` (platform-appropriate) or `fseek`/`fread` for positional reads.

## Phase 2 — Multithreading (All Methods)

**Every function that reads or writes chunks must accept `int threads = 0` (0 =
use all available threads via `omp_get_max_threads()`).**

```cpp
// Helper used everywhere:
inline int resolve_threads(int threads) {
    return (threads <= 0) ? omp_get_max_threads() : threads;
}
```

### 2.1 Write parallelism

`Rcpp_sp_write` (and the underlying writer) should encode chunks in parallel:
- Allocate chunk output buffers sized by worst-case
- `#pragma omp parallel for schedule(dynamic) num_threads(nthreads)` over chunk indices
- Write sequentially after all are encoded (order matters for file layout)

Add `int threads = 0` parameter to `Rcpp_sp_write`, `Rcpp_sp_write_dense`, and
all writer C++ entry points.

### 2.2 Read parallelism

`Rcpp_sp_read` and `Rcpp_sp_read_transpose` should decode chunks in parallel:
- Read all chunk descriptors first
- Parallel decode: each thread decodes its assigned chunks independently
- `pread` (Linux/macOS) / `ReadFile` with `OVERLAPPED` (Windows) for thread-safe
  concurrent reads without seeking (no mutexes needed)
- Accumulate into pre-allocated output arrays

**Windows note**: Use `#ifdef _WIN32` guards around `pread` vs. `ReadFile`. Provide
a `safe_pread()` inline wrapper in a new file
`inst/include/streampress/core/platform_io.hpp` that abstracts both.

```cpp
// inst/include/streampress/core/platform_io.hpp
#ifdef _WIN32
#  include <windows.h>
static inline ssize_t safe_pread(int fd, void* buf, size_t count, uint64_t offset) {
    HANDLE h = (HANDLE)_get_osfhandle(fd);
    OVERLAPPED ov = {};
    ov.Offset     = (DWORD)(offset & 0xFFFFFFFFULL);
    ov.OffsetHigh = (DWORD)(offset >> 32);
    DWORD nread = 0;
    if (!ReadFile(h, buf, (DWORD)count, &nread, &ov)) return -1;
    return (ssize_t)nread;
}
#else
#  include <unistd.h>
static inline ssize_t safe_pread(int fd, void* buf, size_t count, uint64_t offset) {
    return pread(fd, buf, count, (off_t)offset);
}
#endif
```

Use `safe_pread` everywhere in the reader. This gives Linux, macOS, and Windows
support without platform-specific code paths in business logic.

### 2.3 Thread count in R API

Add `threads = 0L` to every `st_*` R function that does I/O:
- `st_write(..., threads = 0L)`
- `st_read(..., threads = 0L)`

## Phase 3 — Memory-Driven Chunk Sizing

### 3.1 Logic

The user should be able to specify chunk size in **bytes** (default: 64 MB) or
as an explicit column count. Add these parameters to `st_write`:

```r
st_write(x, path,
         chunk_cols  = NULL,     # explicit integer column count, or NULL
         chunk_bytes = 64e6,     # target bytes per chunk (used if chunk_cols is NULL)
         ...)
```

At the R level:
```r
if (is.null(chunk_cols)) {
    bytes_per_col <- nrow(x) * 4L  # float32 = 4 bytes worst case
    chunk_cols <- max(1L, as.integer(floor(chunk_bytes / bytes_per_col)))
    chunk_cols <- min(chunk_cols, ncol(x))   # cap at actual column count
}
chunk_cols <- as.integer(chunk_cols)
```

### 3.2 Transpose chunk sizing

The transpose section may have a different natural chunk size. For a matrix with
`m` rows and `n` columns:
- Forward chunks cover `chunk_cols` columns → each is `m × chunk_cols` cells
- Transpose chunks cover rows of A (= columns of Aᵀ) → to match the same memory
  footprint, transpose `chunk_cols_T ≈ chunk_bytes / (n * 4)` columns of Aᵀ

Add a `transp_chunk_bytes` (or `transp_chunk_cols`) parameter to `st_write`:

```r
st_write(x, path,
         chunk_cols        = NULL,
         chunk_bytes       = 64e6,
         transp_chunk_cols = NULL,    # explicit, or NULL to use chunk_bytes
         transp_chunk_bytes = NULL,   # NULL = use chunk_bytes for transpose too
         include_transpose = TRUE,
         ...)
```

If both `transp_chunk_cols` and `transp_chunk_bytes` are NULL, use `chunk_bytes`
for transpose sizing as well (results in a different column count for Aᵀ when
`m != n`). Store `transp_chunk_cols` in `hdr.set_transp_chunk_cols()`.

## Phase 4 — Row Slicing via Transpose

### 4.1 Add `include_transpose = TRUE` as the new default in `st_write`

The old default was `FALSE`. Change to `TRUE` so all new writes include a
transpose section. Keep the parameter so users can opt out.

### 4.2 Implement `Rcpp_st_slice_rows`

```cpp
// [[Rcpp::export]]
S4 Rcpp_st_slice_rows(const std::string& path,
                      const IntegerVector& rows,   // 1-indexed R row indices
                      int threads = 0);
```

Algorithm:
1. Open file, read header
2. Check `hdr.transpose_offset != 0`. If 0, throw: "No transpose section in file.
   Re-write with st_write(..., include_transpose=TRUE)."
3. Read transpose chunk index (stored at transpose_offset)
4. Map requested rows → which transpose chunks contain them
   (each transpose chunk covers `transp_chunk_cols` rows of Aᵀ = columns of Aᵀ)
5. Parallel decode of the needed transpose chunks using `safe_pread`
6. Extract requested rows from decoded columns, assemble output `dgCMatrix`
   (rows of output = requested rows, cols of output = all n columns)

Return value: `dgCMatrix` of size `length(rows) × n`.
If `nnz > 2^31 - 1`, split into list of dgCMatrix along columns and return as R
list with attribute `"col_starts"` giving the starting column of each element.

### 4.3 Add `st_slice_cols` as a proper exported function

Currently partial column reads exist but are bundled in `st_read(cols=)`. Add:

```r

st_slice_cols <- function(path, cols, threads = 0L) {
  path <- normalizePath(path, mustWork = TRUE)
  Rcpp_st_slice_cols(path, as.integer(cols), as.integer(threads))
}
```

Implement `Rcpp_st_slice_cols` analogously to `Rcpp_sp_read` with the `cols`
parameter, but with parallel chunk decoding. Remove the redirect through
`Rcpp_sp_read(..., cols=cols)`; give it a dedicated implementation.

### 4.4 Convenience combination

```r
#' @export
st_slice <- function(path, rows = NULL, cols = NULL, threads = 0L) {
  path <- normalizePath(path, mustWork = TRUE)
  if (!is.null(rows) && !is.null(cols)) {
    # First slice columns (fast), then filter rows from result in memory
    A <- st_slice_cols(path, cols, threads = threads)
    return(A[rows, , drop = FALSE])
  } else if (!is.null(rows)) {
    return(st_slice_rows(path, rows, threads = threads))
  } else if (!is.null(cols)) {
    return(st_slice_cols(path, cols, threads = threads))
  } else {
    return(st_read(path, threads = threads))
  }
}
```

## Phase 5 — Chunk Iteration

The user wants to process one chunk at a time (e.g. for streaming pipelines). Use
a stateless design: expose chunk boundaries so the user can iterate manually.

### 5.1 `st_chunk_ranges`

```r
#' Return the column ranges for each chunk in a .spz file
#'
#' @param path Path to a .spz file.
#' @return A data.frame with columns \code{start} and \code{end} (1-indexed,
#'   inclusive), one row per chunk.
#' @export
st_chunk_ranges <- function(path) {
  info <- st_info(path)
  chunk_cols <- info$chunk_cols
  n <- info$cols
  starts <- seq(1L, n, by = chunk_cols)
  ends   <- pmin(starts + chunk_cols - 1L, n)
  data.frame(start = starts, end = ends)
}
```

No new C++ code needed — uses existing `st_info()`.

### 5.2 `st_map_chunks`

For parallel application of a function to every chunk:

```r
#' Apply a function to every chunk in a .spz file
#'
#' @param path Path to a .spz file.
#' @param fn Function taking \code{(chunk, col_start, col_end)} where
#'   \code{chunk} is a \code{dgCMatrix}.
#' @param transpose Logical; if TRUE, iterate over transpose chunks (row chunks
#'   of the original matrix). Requires transpose section in file.
#' @param threads Integer; decode threads per chunk. Default 0 (all threads).
#' @return Invisible list of results from \code{fn}.
#' @export
st_map_chunks <- function(path, fn, transpose = FALSE, threads = 0L) {
  ranges <- st_chunk_ranges(path)
  results <- vector("list", nrow(ranges))
  if (!transpose) {
    for (i in seq_len(nrow(ranges))) {
      chunk <- st_slice_cols(path, ranges$start[i]:ranges$end[i], threads = threads)
      results[[i]] <- fn(chunk, ranges$start[i], ranges$end[i])
    }
  } else {
    info <- st_info(path)
    tc <- if (info$transp_chunk_cols > 0L) info$transp_chunk_cols else info$chunk_cols
    row_starts <- seq(1L, info$rows, by = tc)
    row_ends   <- pmin(row_starts + tc - 1L, info$rows)
    results <- vector("list", length(row_starts))
    for (i in seq_along(row_starts)) {
      chunk <- st_slice_rows(path, row_starts[i]:row_ends[i], threads = threads)
      results[[i]] <- fn(chunk, row_starts[i], row_ends[i])
    }
  }
  invisible(results)
}
```

## Phase 6 — Metadata-Based Filtering

Convenience wrappers that read obs/var, filter, then slice:

```r
#' Read obs metadata, apply filter expression, return matching row indices
#' @param path Path to .spz file
#' @param ... Named filter criteria passed to \code{subset()} on the obs table
#' @return Integer vector of matching row indices (1-based)
#' @export
st_obs_indices <- function(path, ...) {
  obs <- st_read_obs(path)
  if (nrow(obs) == 0L) stop("File has no obs table")
  matched <- subset(obs, ...)
  which(rownames(obs) %in% rownames(matched))
}

#' Slice rows matching obs filter criteria
#' @param path Path to .spz file
#' @param ... Filter expression on obs columns (passed to subset())
#' @param threads Integer decode threads. 0 = all.
#' @return dgCMatrix
#' @export
st_filter_rows <- function(path, ..., threads = 0L) {
  idx <- st_obs_indices(path, ...)
  if (length(idx) == 0L) stop("No rows match filter criteria")
  st_slice_rows(path, idx, threads = threads)
}

#' Slice columns matching var filter criteria
#' @param path Path to .spz file
#' @param ... Filter expression on var columns (passed to subset())
#' @param threads Integer decode threads. 0 = all.
#' @return dgCMatrix
#' @export
st_filter_cols <- function(path, ..., threads = 0L) {
  var <- st_read_var(path)
  if (nrow(var) == 0L) stop("File has no var table")
  matched <- subset(var, ...)
  idx <- which(rownames(var) %in% rownames(matched))
  if (length(idx) == 0L) stop("No columns match filter criteria")
  st_slice_cols(path, idx, threads = threads)
}
```

## Phase 7 — List Assembly (Streaming cbind Write)

Allow streaming assembly of multiple matrices into one .spz:

```r
#' Write a list of matrices as a single column-concatenated .spz file
#'
#' Matrices are cbind-assembled. All matrices must have the same number of rows.
#' An obs/var table may be provided for the combined matrix.
#'
#' @param x A list of dgCMatrix objects (or objects coercible to dgCMatrix).
#'   All must have identical \code{nrow}.
#' @param path Output .spz path.
#' @param obs data.frame with cell metadata (nrow == total columns across all x).
#' @param var data.frame with gene metadata (nrow == nrow of each matrix in x).
#' @param chunk_bytes Target bytes per chunk. Default 64 MB.
#' @param chunk_cols Explicit column count per chunk. Overrides chunk_bytes.
#' @param include_transpose Logical. Default TRUE.
#' @param precision Value precision. Default "auto".
#' @param threads Integer. 0 = all threads.
#' @param verbose Logical.
#' @return Invisibly, a list of compression statistics.
#' @export
st_write_list <- function(x, path, obs = NULL, var = NULL,
                          chunk_bytes = 64e6, chunk_cols = NULL,
                          include_transpose = TRUE, precision = "auto",
                          threads = 0L, verbose = FALSE) {
  stopifnot(is.list(x), length(x) >= 1L)
  mats <- lapply(x, function(m) {
    if (!inherits(m, "dgCMatrix")) m <- as(m, "dgCMatrix")
    m
  })
  nr <- nrow(mats[[1]])
  if (!all(sapply(mats, nrow) == nr)) stop("All matrices must have the same nrow")
  combined <- do.call(cbind, mats)
  st_write(combined, path, obs = obs, var = var,
           chunk_bytes = chunk_bytes, chunk_cols = chunk_cols,
           include_transpose = include_transpose, precision = precision,
           threads = threads, verbose = verbose)
}
```

For very large lists where cbinding exhausts RAM, a streaming writer is ideal but
complex. For now `do.call(cbind, mats)` is acceptable; document the RAM requirement
and leave a code comment noting a future streaming version.

## Phase 8 — Wire Dimnames to Returned Objects

Currently `Rcpp_sp_read` returns a bare `dgCMatrix` without dimnames even when
ROWNAMES/COLNAMES metadata is present in the file. Fix this at the R level:

In `st_read` (and `st_slice_cols`, `st_slice_rows`, `st_slice`), after calling
the Rcpp function:

```r
st_read <- function(path, cols = NULL, threads = 0L, reorder = TRUE) {
  path <- normalizePath(path, mustWork = TRUE)
  A <- Rcpp_sp_read(path, cols = cols, reorder = reorder, threads = as.integer(threads))
  # Wire dimnames from metadata
  info <- Rcpp_sp_metadata(path)
  if (!is.null(info$rownames) && length(info$rownames) == nrow(A))
    rownames(A) <- info$rownames
  if (!is.null(info$colnames)) {
    cn <- info$colnames
    if (!is.null(cols)) cn <- cn[cols]
```
